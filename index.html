Los eventos onmouseover y onmouseout y onclick

Las onmousedown, onmouseupy onclicklos acontecimientos son todos partes de un clic del ratón. Primero, cuando se hace clic en un botón del mouse, se activa el evento onmousedown, luego, cuando se suelta el botón del mouse, se activa el evento onmouseup, finalmente, cuando se completa el clic del mouse, se activa el evento onclick.

¡Una HTMLCollection NO es una matriz!
Una colección HTMLCollection puede parecer una matriz, pero no lo es.
Puede recorrer la lista y hacer referencia a los elementos con un número (como una matriz).
Sin embargo, no puede usar métodos de matriz como valueOf (), pop (), push () o join () en una HTMLCollection.

Event Listener

El event listener es un método que controla cuando, cómo y en dónde se va a dar un elemento especificado por el usuario

Se pueden añadir varios controladores de eventos a un mismo elemento sin sobreecribirlos

Se usan dos event listeners:

  • addEventListener(): Para añadir un controlador de eventos.
  • removeEventListener(): Para eliminar un controlador de eventos.

Sintaxis para los event listener

elemento.addEventListener(evento, función, useCapture)
  • evento: Para el tipo de evento con el que se quiera accionar la acción.
  • función: Para la función que se va a ejecutar una vez que se de el evento.
  • useCapture: Es un parámetro opcional en el cual se especifica si se va a usar una captura de eventos o un burbujeo de eventos.

DOM Navigation

Sirve para navegar dentro de todos los elementos de un documento HTML.

Nodos DOM

Todo dentro de un documento HTML es un nodo, cada elemento, texto, cada atributo y todos sus comentarios.

Relaciones de nodo

Un node tree tiene relaciones jerárquicas, los padres, hijos y hermanos son estas relaciones, siendo el nodo superior la llamada raíz

Cada nodo tiene un padre (exceptuando la raíz), un nodo padre puede tener varios hijos y los hermanos son aquellos nodos que vienen del mismo padre.
